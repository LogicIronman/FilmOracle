# FilmOracle 最终验收、报告评析与分析计时设计

## 目标

以 Obsidian `开发/FilmOracle/题目要求.md` 为唯一题目基线完成最终验收，并修复两项已复现问题：导出报告把结构化评析输出为 JSON，以及评论分析过程没有实时耗时显示。

## 验收范围

- 评论抓取、单条或批量导入、按电影分类管理。
- AI 批量情感分析、关键词提取、好评率和情感图表。
- 关键词与情感类型筛选。
- Servlet、JDBC、JSP、Git 功能分支、Docker Tomcat/MySQL、导入卷和资源限制。
- 自动化测试、WAR 构建及关键页面流程。

## 报告评析格式

新增纯函数 `formatReviewHtml(review)`，统一把评析转换为安全 HTML：

- 对象格式依次显示总体评价、出彩、不足、最终判断和完整评析。
- JSON 字符串先解析为对象；普通字符串按纯文本显示。
- 空值显示“暂无评析”。
- 所有动态文本必须经过 `escapeHtml`，不得把对象隐式转换为 `[object Object]` 或 JSON 源文输出。
- 页面评析与导出报告复用同一函数，避免两套格式继续漂移。

## 分析计时

新增独立计时状态，提供 `startTaskTimer`、`stopTaskTimer`、`resetTaskTimer` 和 `formatElapsedTime`：

- 爬取并本地分析、爬取并 AI 分析、导入评论本地分析、导入评论 AI 分析开始时启动。
- 任务头部状态实时显示 `状态 · 00:12.3`。
- 成功或失败时停止并冻结耗时，同时在任务日志追加 `总耗时: 12.3 秒`。
- 新任务启动前清理旧 interval，避免重复计时器和内存泄漏。
- 使用 `performance.now()`，不可用时回退到 `Date.now()`。

## 测试与验收

- Node 测试验证对象、JSON 字符串、普通字符串和空评析的格式化结果。
- Node 测试验证耗时格式和源码中的启动/停止接线。
- 运行全部 JS 测试、`mvn test`、`mvn clean package`。
- 最终按题目逐条给出符合、部分符合和缺失/风险清单。

## 边界

- 不新增独立评论管理页面；题目中的“单条录入或批量导入”已由批量导入界面及评论 API 覆盖。
- 不重写现有 JSP 架构；保留 `index.jsp` 作为服务端入口并复用现有页面。
- 不修改与本轮无关的情感分类、图表或缓存逻辑。
