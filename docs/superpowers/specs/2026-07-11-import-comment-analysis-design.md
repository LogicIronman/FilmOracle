# 导入评论专用分析设计

## 目标

将评论文件导入改为独立分析流程：本地规则和 AI 都只能使用文件内实际解析出的评论，不能请求或混入豆瓣评论。

## 范围

- 只修改导入评论页面、导入接口以及导入后的分析调用。
- 不调整自然搜索、电影详情搜索流程或其他页面的功能。
- 保持 Java Servlet、MySQL、原生 JavaScript/CSS 技术栈。

## 导入页面

导入页标题改为“导入评论分析”，保留电影信息输入与文件选择，并在页面头部新增“文件规格”按钮。页面提供两个明确入口：

- `导入并本地分析`：上传、解析、保存导入文件中的评论后，在浏览器运行本地规则分析。
- `导入并 AI 分析`：上传、解析、保存同一批导入评论后，调用现有 `/api/analyze`；现有 MySQL AI 缓存照常按电影、完整评论、模型和提示词命中。

电影标题用于一次性查询豆瓣电影信息以补全简介；查询失败时简介保持用户输入值或空值。此查询不得获取评论。

## 文件规格页面

新增独立 `import-spec` 视图，说明：仅支持 UTF-8 编码的 `.txt` / `.csv`；评论文本必填；CSV 可选 `rating` / `star` 和 `user` / `name` 列；TXT 每行一条评论，可用 `5星 评论内容` 标记星级；空行和少于两个字符的评论忽略。页面提供返回导入页的入口和可复制示例。

## 数据流

1. 前端用 `FormData` 将电影信息与文件提交到 `/api/comments/import`。
2. 后端在该接口解析文件、持久化评论，并在响应中返回这批解析后的评论和标准化电影信息。
3. 前端将响应评论赋给 `currentComments`；本地分析调用 `buildAnalysis(currentComments, movie)`。
4. AI 分析仅向 `/api/analyze` 发送同一批 `currentComments`，不调用 `loadCommentsAndAnalyze`，也不调用 `/api/movie/.../interests`。

## 错误处理与验证

- 无文件、错误扩展名、无有效评论时阻止分析并保留在导入页显示原因。
- 后端测试导入解析后的响应包含原始导入评论。
- 前端语法检查、Maven 全量测试、Docker 实测 TXT/CSV 导入后本地和 AI 分析均不请求豆瓣评论。
